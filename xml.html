Cos'è l'XML?


XML ( Extensible Markup Language ) è un linguaggio che consente la rappresentazione di documenti e dati strutturati su supporto digitale. È uno strumento potente e versatile per la creazione, memorizzazione e distribuzione di documenti digitali. L'XML è un linguaggio di markup: esso infatti è composto da istruzioni, definite tag o marcatori, che descrivono la struttura e la forma del documento. Ogni marcatore (o coppia di marcatori) identifica un elemento o componente del documento.


Di seguito è presente un esempio di un documento XML: 

  
    <documento>  <!-- Marcatore di inzio -->
      <corpo>
        Testo di esempio
      </corpo>
    </documento>  <!-- Marcatore di fine -->


Attenzione!

XML è case sensitive: scrivere <documento> è diverso da scrivere <Documento>.




Struttura di un documento XML


Un documento XML è formato da una struttura logica e una struttura fisica.



Struttura logica


Un documento XML è strutturato in modo gerarchico ed è composto da elementi. Un elemento è un componente logico del documento e può a sua volta contenere testo o altri sotto-elementi. Gli elementi sono organizzati ad albero con radice root e ad esso possono essere associati delle informazioni descrittive chiamate attributi.



Struttura fisica


La struttura di un documento XML viene rappresentata mediante marcatori ( markup ). Gli elementi sono rappresentati mediante tag, ovvero coppie di marcatori che racchiudono il contenuto dell’elemento. I sottoelementi sono tag contenuti all’interno di un altro tag. Gli attributi vengono rappresentati sotto forma di coppie nome-valore all’interno dei tag mentre la radice è un tag che racchiude tutto il resto del documento.




Struttura logica e fisica


Esiste una corrispondenza diretta fra struttura fisica e struttura logica ( tree )

Di seguito è presente lo "scheletro" di un documento XML: 

  
    <root>
      <child>
        <subchild>
          ...
        </subchild>
      </child>
      <child>
        ...
      </child>
    </root>



Struttura formale di un documento XML


Un documento è costituito da due parti:

  • Prologo: contiene una dichiarazione XML ed il riferimento ( opzionale ) ad altri documenti che ne definiscono la struttura o direttive di eleborazione.
  • Corpo: è il documento XML vero e proprio.


    •   
    <?xml version="1.0" encoding="UTF-8">  <!-- Prologo -->
    <root>
      <child>
        ...
      </child>
      <child>
        ...
      </child>
    </root>



    Elementi e tag


    Un elemento è un frammento di testo racchiuso fra uno start tag e un end tag:

  • Lo start tag è costituito da un nome più eventuali attributi racchiusi dai simboli ' < ' e ' > '.
  • L'end tag è costituito da un nome (lo stesso dello start tag) racchiuso da ' </ ' e ' > '.



    • Attenzione!

    I tag non possono avere nomi che iniziano per XML, XMl, Xml, xml...




    Attributi


    A ogni elemento possono essere associati uno o più attributi che ne specificano ulteriori caratteristiche ( la lingua del suo contenuto testuale, un identificatore univoco, un numero di ordine, ... ). Essi sono caratterizzati da un nome che li identifica e un valore. 


      
    <articolo titolo="Titolo dell'articolo"/>



    Validazione di un documento XML: che cos'è DTD?


    DTD ( Document Type Definition ) è un linguaggio per definire la grammatica che descrive la composizione degli elementi costituenti una certa classe di documenti XML e fornisce uno strumento per la validazione.




    Dichiarazione del DTD


    Per applicare un DTD ad un documento XML nel suo prologo ( sotto l’XML Declaration ) dobbiamo inserire una dichiarazione con questa sintassi: 


      
    <!DOCTYPE root-element SYSTEM "filename">

    Dove:

  • root-element è il nome dell’elemento radice.
  • SYSTEM definisce documenti di utilizzo locale.
  • filename è il file che contiene il DTD.




    • Esempio


    Ecco un esempio di un file XML con il suo relativo DTD. 


      
    <!-- XML -->
    <?xml version="1.0" encoding="UTF-8">
    <!DOCTYPE message SYSTEM "message.dtd">
    <message>
      <to>Bob</to>
      <from>Janet</from>
      <heading>Remineder</heading>
      <body>Don't forget me this weekend</body>
    </message>

    <!-- DTD -->
    <!ELEMENT message (to,from,heading,body)>
    <!ELEMENT to (#PCDATA)>
    <!ELEMENT from (#PCDATA)>
    <!ELEMENT heading (#PCDATA)>
    <!ELEMENT body (#PCDATA)>






    Parsing di un documento XML


    Ci sono diversi metodi per eseguire il parsing di un documento XML: il più comune ed il più utilizzato è l'utilizzo della funzione fetch().
    La funzione fetch(), simile ad una promise, è una funzione ASINCRONA ( il codice viene eseguito in parallelo con le altre operazioni ). La fetch() scarica tutto il contenuto della pagina web ( HTML, CSS, JSON, ecc... ), per questo richiede un parametro: l'URL.

    La fetch() utilizza due funzioni chiamate .then():

  • Il primo .then() viene richiamato dopo lo scaricamento di tutti i dati della pagina e all'interno è presente la variabile che contiene tutti i dati scaricati come testo.
  • Il secondo .then() viene richiamato dopo l'esecuzione del primo .then() e serve creare un oggetto di tipo DOM Parser ( che si trova nel browser ) dove viene salvato l'XML, mentre con il comando parseFromString viene parsato l'XML. Il parseFromString richiede 2 parametri: il primo parametro è la variabile che contiene i dati scaricati, mentre il secondo è "application/xml".


    •   
    const URL = "your_url"
    fetch(URL)
    .then(response => response.text())
    .then(data => {
      let obj_dom = new DOMParser();
      let parseXML = obj_dom.parseFromString(data,"application/xml")
      console.log(parseXML)
    })
